# 05 - Scene Map

Competitive/strategic intelligence on the DayZ modding and server-ops community. Current as of early 2026. No sanitizing.

---

## Top modders and their signatures

### **DayZ-Expansion team** (LieutenantMaster, Jacob_Mango, Salutesh, et al.)
- **Makes:** [DayZ-Expansion](https://steamcommunity.com/workshop/filedetails/id=2572331007) - the kitchen-sink gameplay overhaul. Helicopters, cars, missions, airdrops, base-building, safezones, territories, markets, AI. Modular - servers pick pieces - but most run the whole bundle.
- **Reputation:** The industry standard for modded gameplay. If a server is "modded," assume Expansion is on it. Active dev through 2025-2026 ([GitHub](https://github.com/ExpansionModTeam)).
- **Where:** [Discord discord.gg/rMZuS4F](https://discord.gg/rMZuS4F). Professional, well-moderated. Files DMCA on reuploads.

### **Arkensor** (+ collaborators on **Community Framework / CF**)
- **Makes:** [Community Framework](https://github.com/Arkensor/DayZ-CommunityFramework). Foundational layer: RPC manager, modules, notifications, UI extensions, NetSync vars. **Almost every script mod in the scene depends on CF.**
- **Reputation:** Semi-official. [Arkensor now responds to bugs on Bohemia's Feedback Tracker](https://feedback.bistudio.com/p/Arkensor/) with what looks like internal testing access. Read: CF is effectively sanctioned infrastructure, not a pure community project.
- **Where:** GitHub. DayZ Modders Discord.

### **Sumrak / Adam Franců** - **Namalsk** creator, now Creative Lead at Bohemia for DayZ ([Exitlag](https://www.exitlag.com/blog/dayz-namalsk-map/))
- **Makes:** [Namalsk Island](https://steamcommunity.com/workshop/filedetails/id=2289456201) + [Namalsk Survival](https://steamcommunity.com/workshop/filedetails/id=2289461232). The gold standard for a community map. Abandoned Cold War island, EVR storms, blizzards, frostbite, underground. Custom events, POIs, soundtrack. "Plays like an official map."
- **Reputation:** Universally respected. Zero drama. Explicitly forbids monetization: _"You cannot monetize this content, including selling priority queue slots. Any re-uploads will be taken down via DMCA notice without warning."_ ([Changelog](https://steamcommunity.com/sharedfiles/filedetails/changelog/2289461232))
- **Where:** [GitHub SumrakDZN](https://github.com/SumrakDZN/Namalsk-Server), [nightstalkers.cz](https://nightstalkers.cz/).

### **Jacob_Mango**
- **Makes:** Co-creator of CF; core Expansion contributor; co-authored the first post-0.63 modded weapons pack.
- **Reputation:** Prolific and foundational. One of the highest-velocity names in the scene.
- **Where:** GitHub, Expansion Discord, Bohemia Feedback tracker.

### **Dr_J0nes** - **Trader** mod
- **Makes:** [Trader](https://steamcommunity.com/sharedfiles/filedetails/id=1590841260) - AI traders, safe zones, currency. Most modded servers that have economy run Dr_Jones Trader or the Expansion market.
- **Reputation:** Adoption is huge; maintenance is spotty. After every DayZ patch there's a few days/weeks where Trader is broken and community fixes circulate ([Steam Workshop page](https://steamcommunity.com/sharedfiles/filedetails/id=1590841260)). No dedicated Discord anymore - support happens in the DayZ Modders Discord. Strict "DO NOT REUPLOAD" license.
- **Where:** Scattered. [Scalespeeder's GitHub](https://github.com/scalespeeder) maintains config templates per DayZ version.

### **DaemonForge** - **UniversalAPI** creator
- **Makes:** [DayZ-UniveralApi](https://github.com/DaemonForge/DayZ-UniveralApi) - the de-facto external data webservice for DayZ. MongoDB-backed, Mongo-ish query/save/load/update API over HTTP. Ships with Enforce SDK. Adopted by MapLink Hive and numerous private server networks.
- **Reputation:** Respected but quiet. He's the guy you'd be competing with if you build the "external data service" category. Extensibility is the moat: it works, people use it, switching cost is real.

### **Other heavy hitters**
- **Kaffeina** - [NatureOverhaul](https://steamcommunity.com/sharedfiles/filedetails/id=2875882501) (overgrown Chernarus, dystopian decay).
- **HennySea + team** - [DayZ-Dog](https://steamcommunity.com/sharedfiles/filedetails/id=2792982069).
- **Staiff** - No Sprinting Zombies (Romero-style infected).
- **Pattow** - zSpawnSelection.
- **Windstride** - Clothing Pack.
- **BaseBuildingPlus team** - [BaseBuildingPlus](https://steamcommunity.com/sharedfiles/filedetails/id=1828439124). Had a multi-year maintenance gap; updated for 1.27 in 2025.
- **Repackaging defense** - Some established modders preemptively DMCA repackagers to protect their work.

## The Discords that actually matter

### Core modding (Enforce + tooling)
- **DayZ Modding Community (D.Z.M.C)** - [discord.com/invite/eRpp9Mg](https://discord.com/invite/eRpp9Mg). ~7,000 members. Active, beginner-friendly, non-toxic. The default place to lurk if you're learning Enforce.
- **Tater's Modding Community** - Smaller, more hands-on. Good for specific technical Enforce questions.

### Server operations
- **Official DayZ Discord** - [discord.com/invite/dayz](https://discord.com/invite/dayz). 170k+. Big, Bohemia-official, not modding-focused. Good for update announcements and policy pulse.
- **DAYZ X** - [discord.com/invite/dayzconsole](https://discord.com/invite/dayzconsole). 20-server network, console-focused. Running AI-powered support, custom NPCs, etc. - they're one of the bigger console-modded organizations.
- **CFTools / Omega Manager Discord** - Go here if you're running a serious PC server.

### Specific big mods
- **DayZ-Expansion** - discord.gg/rMZuS4F. Well-run.
- **Dr Jones Trader** - no dedicated Discord anymore (support folded into D.Z.M.C).

### Console modding
- **Bennett's DayZ Modding** - [discord.com/invite/bennetts-dayz-modding](https://discord.com/invite/bennetts-dayz-modding). ~4,800 members. The major console-modding hub. Parent org claims ~7k across its Discords and ~500k lifetime players.

### High-quality communities (to study, not spam)
- **DayZ Underground (DUG)** - [dayzunderground.com](https://www.dayzunderground.com/). 128-slot FPP hardcore RP. Whitelist. Cliquey. High production value. Worth lurking for how a top-tier RP community structures itself.
- **DayZ n' Chill (DZNC)** - 7-year community, open membership.

### Red flags / caveats
- **OpenDayZ forums** - legacy, some spam infiltration, unclear moderation in 2025.
- **Steam Workshop comments** - toxic and unreliable. DMCA-abuse drama makes it hostile.

## Current hot topics & drama (2024-2026)

### Bohemia DDoS attack (Feb 2025)
Russian hacking group "Xiangjang_zhi" targeted DayZ + Arma Reforger servers, demanded >$78k in Bitcoin (later called it a "joke"). Continued over a week. Bohemia said they were "making progress" ([TheGamer](https://www.thegamer.com/arma-reforger-dayz-ddos-attack-continues-bohemia-interactive-issues-statement/)). Community did not rally to the hackers. Relevance: Bohemia infrastructure is under active geopolitical attack; operational resilience is on the org's mind.

### DMCA-abuse extortion scheme (ongoing)
Some actors file bulk bogus DMCAs to take down popular DayZ mods ([Steam forum](https://steamcommunity.com/discussions/forum/7/3758849893337016999/), [NotebookCheck](https://www.notebookcheck.net/DMCA-abuse-on-Steam-is-out-of-control-and-Valve-isn-t-fixing-it.1084373.0.html)). Target mod pulled for 10 days while counter-notice processed. Counter-notice requires the server owner to submit real name + phone to Valve. **This is a real risk for any popular mod:** filing a counter-notice means disclosing your legal identity, so plan for it before you ship something widely adopted. Legitimate modders are now using tools like "Sqi Workshop Crawler" to find repackaged versions; reports of [5,000+ infringement hits on just 2 mods](https://steamcommunity.com/discussions/forum/1/4202490652503169554/).

### Monetization crisis
Bohemia's monetization program expires **31 Jan 2027**. Unclear if it gets renewed, replaced, or tightened. Server networks running Tebex / Patreon setups selling "VIP" loadouts, banking interest, and base-building perks are in a permanent grey zone ([Nexus Games guide](https://nexus-games.com/us/blog/how-to-create-profitable-dayz-server-community-donations/)). Critics call it pay-to-win; Bohemia looks the other way so long as nobody's obviously gating core gameplay. A user complaint: ["mod dev making you pay monthly to get the mod"](https://steamcommunity.com/app/221100/discussions/0/3802775659503812474/) - some mods literally paywall themselves.

### 1.29 update broke mods (2025)
Expansion and CF initially broke on 1.29; Bohemia extended the experimental period 2+ months to let modders catch up ([Steam discussion](https://steamcommunity.com/app/221100/discussions/0/600766503721107730/)). Community sentiment: DayZ patches are mod-hostile; rolled Central Economy changes (planned scarcity overhaul) got scrapped because too many mods depended on the old surface.

### DayZ 2 trademark (April 2026)
Bohemia filed with EUIPO ([Playfront](https://playfront.de/en/dayz-2-ist-offiziell-in-arbeit-bohemia-meldet-markenschutz-an/)). Successor is in planning. Practical implication: current Enfusion engine is "legacy" and [future Enfusion games won't carry legacy modules](https://www.bohemia.net/blog/Introducing-Enfusion-Engine) - meaning DayZ 1.x-trained modders will have a head start on DayZ 2 modding but will need to relearn pieces.

### Vanilla feels empty
Recurring refrain: [DayZ lacks depth after a decade](https://www.sportskeeda.com/esports/7-best-dayz-mods-try-2025). [70%+ of servers feel like copies of each other](https://steamcommunity.com/app/221100/discussions/0/4034727291139710032/) - PvE against dumb AI, hoarders in unraidable bases, traders, loot stacking. Finding "Vanilla Plus" (depth without departing from core tension) is hard. Community servers and mods keep the game alive more than Bohemia's updates do.

### Code-quality complaints
["One broken mod can take a server down for days"](https://steamcommunity.com/app/221100/discussions/0/600766503721107730/) - stacking 20+ mods creates brittleness. Cut-and-paste of broken code proliferates. "Less is more" sentiment is rising among serious server ops.

### Individual modder feuds
Mostly happens inside Discords and Steam Workshop comment threads. The biggest public beef patterns: (1) repackagers vs. authors, (2) competing mods in the same category trading DMCA threats, (3) drama over whose fork of a deprecated mod becomes "the" community fork.

## What the community is begging for and nobody has built

These are the recurring "I wish someone would make X" asks I found across Reddit, modding Discords, Steam forums, and modder-guide blog posts:

### 1. World immersion / "lived-in" feel
_"Make houses look like they weren't never lived in."_ Furniture variety, personal effects (photos, books, clothes), looting evidence, overgrown gardens, toys, graffiti, barricaded windows. Chernarus is a mall compared to what players imagine it could be. Mostly cosmetic work but requires a lot of asset authoring and placement.

### 2. Deep hunting / ecology
Existing hunting is shallow. Players want: dynamic populations responsive to pressure, tracking mechanics (blood trails, prints), herd/migration AI, breeding seasons, butchering depth. Some pieces exist (Radioactive Animals) but the full systems mod doesn't.

### 3. Dynamic events & procedural quests beyond airdrops
Expansion Missions is a framework, not a content library. Most RP servers hand-author events. Players want: procedural quest chains, faction wars with persistent territory consequences, environmental disasters (fires, floods, blizzards).

### 4. Plug-and-play cross-server persistence
MapLink Hive exists but is ["by far the hardest mod to get working"](https://steamcommunity.com/sharedfiles/filedetails/id=3478983598) - requires UAPI, DB setup, in-game terminal placement, mod-compatibility hell. **A GUI-driven, one-click cross-server hive would meet a clear unmet need.** This is the most obvious architectural gap.

### 5. Advanced medical systems
Medical Attention Extended exists. Players want more: field surgery, sutures, bullet extraction, long-term disabilities (limps, hearing loss), mental health mechanics (PTSD, morale).

### 6. Better vehicle persistence/repair
Vehicles disappear after restarts, require types.xml lifetime tweaks to stay alive, repair is tedious without being realistic. Unsolved.

### 7. Polished Vanilla Plus framework
Depth without departing from core tension - **no traders, no unlimited stamina, no map markers, no party HUD** - but with improved base building, deeper survival, quality-of-life. ["Vanilla Plus servers are hard to find"](https://steamcommunity.com/app/221100/discussions/0/4034727291139710032/). There's a real audience and no dominant product.

### 8. Long-term mod maintenance as a service
Mods go stale between patches. Server downtime ensues. A paid maintenance service or a "mod stewardship" program for community mods would fill a gap - though it bumps up against Bohemia monetization rules in an awkward way.

## Saturated vs. wide-open mod categories

### Saturated (oversupplied, low differentiation return)
- **Weapon / gear packs.** MMG, Morty's, Renegade, a dozen more. Adds CPU overhead.
- **Base building.** BBP, Expansion BB, Basic Territories.
- **Trader / economy.** TraderPlus, Dr. Jones, ATMs, banking.
- **Vehicles.** Helicopters, armored vehicles, motorcycles, boats.
- **Clothing packs.** Windstride and others.
- **Quality-of-life.** Unlimited stamina toggles, party HUDs, spawn selection menus, death markers. Often accused of [ruining tension](https://steamcommunity.com/app/221100/discussions/0/3002173313631238526/).

### Wide-open (underserved)
- **World population / ambient NPC life.** Nobody's built farmers, scavengers, refugees.
- **Deep hunting / ecology.**
- **Procedural events / quest chains.**
- **Polished Vanilla+ framework.**
- **Plug-and-play cross-server hive with GUI setup.**
- **Long-term mod maintenance services.**
- **Alternative console hosting** (Nitrado is the single choice and many ops are unhappy).
- **Proper telemetry / observability for solo admins** (CFTools Cloud is good but expensive / pro-oriented).
- **Server-side AI director** that reacts to player clustering, time of day, recent PvP, etc.

## Architectural flirtations - what the scene tried, adopted, rejected

### External APIs / microservices - **adopted partially**
- **UniversalAPI** is the standard.
- **CRDTN Core** uses REST for shops/auth/anti-reselling.
- **MapLink Hive** builds on UAPI for cross-server state.
- **CFTools Cloud** is a full SaaS dashboard.
- **Why not universal:** Setup complexity. Server admins want zero-config. Every external-API mod bolts on a sub-project of DB management that they don't want.

### Web dashboards / admin panels - **strongly adopted**
- **CFTools Cloud** - industry standard.
- **DayZ Server Manager (mr-guard)** - self-hosted NodeJS alternative.
- **DZconfig** - browser-based XML editor.
- **VPP Admin Tools, ZomBerry, COT** - in-game GUIs.
- **Why adopted:** Shared banlists + live killfeeds + real-time player moderation are must-haves for any community over ~20 players.

### AI-driven systems - **mixed**
- **AI NPCs:** Adopted. [Expansion-AI](https://steamcommunity.com/sharedfiles/filedetails/id=2792982069), [AI Bandits](https://steamcommunity.com/sharedfiles/filedetails/id=3628006769), [Dynamic AI Spawns](https://steamcommunity.com/sharedfiles/filedetails/id=2874589934). PvE combat against AI soldiers for loot is a standard gameplay loop.
- **AI-driven loot economy / event director:** **Rejected/Never built.** No widely adopted mod uses AI to dynamically adjust CE or trigger events. CE is still manual XML. Expansion Missions is a framework for scripted events, not an AI director.
- **Why rejected:** Enfusion's script VM performance, engine limits on AI pathfinding, and the general "less is more" sentiment about mod complexity. Also: there's no obvious API surface in the engine for "AI that reads the whole world state and makes economic decisions" - you'd have to build that plumbing first.

### Cross-server persistence - **partially adopted, very hard**
- **MapLink Hive** exists and works, but the tutorial is notoriously rough and mod-compatibility is fragmented. Only very serious server networks run it.
- **Why not universal:** Ops burden.

### Analytics / telemetry - **strongly adopted**
- **DZM (DayZ Monitor)** - telemetry mod, heatmaps, sessions, anti-cheat.
- **DZloggy** - web log analyzer (weather, market, trader, ATM, territory, AI events, gear, health, damage, cover usage).
- **Host-panel perf metrics** - Nitrado + GSPs expose CPU/RAM graphs.
- Server admins stack all three.

### What **hasn't** been tried at scale
- **Edge-computed / serverless** backend for mod state (Cloudflare Workers, Vercel, Supabase Edge Functions). Everyone builds VPS-hosted Docker stacks.
- **Event-sourced architectures** for server state. Nobody treats the event log as the source of truth; it's always a side-car.
- **Structured, standardized mod-to-mod APIs** beyond what CF gives. Mod-to-mod integration is ad-hoc.
- **Mod discoverability / recommendation** beyond Steam Workshop search - no curated index, no "servers like yours run these mods," no automated compatibility matrix.
- **AI-authored content pipelines** (dialogue for AI NPCs, procedural quest text, event narrative). DayZ scene hasn't touched LLMs in any serious way. Huge gap.

---

## Bottom line for a newcomer

1. **CF, Expansion, Trader, Namalsk, AI NPCs, Traders, CFTools** are the nouns of the scene. You must know them.
2. **The power users are on Discord, not Reddit or forums.** Discords D.Z.M.C, Expansion, CFTools.
3. **Mod authors face DMCA abuse as a real threat.** Plan for it.
4. **Bohemia is heavy-handed on malware/DRM and monetization, permissive on almost everything else.** Stay away from those third rails.
5. **The scene is saturated in surface-level mods (guns, traders, bases) and starved in systemic ones (Vanilla+, world population, ecology, cross-server infra, AI director, analytics for solo admins).**
6. **DayZ 2 is coming.** Build things that port or that have a 2-3 year payback window.
